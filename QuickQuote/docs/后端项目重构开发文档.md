# QuickQuote 后端维护手册

更新日期：2026-06-23

## 1. 当前状态

后端已从旧的动态 SQL/单源思路调整为结构化多源召回。继续维护时，应围绕以下原则增量开发：

- 数据源独立召回。
- 候选统一合并。
- 来源字段保留。
- 前端字段尽量稳定。
- 单源失败不影响整批结果。

## 2. 新增数据源建议

新增数据源时建议提供服务类：

```text
app/services/xxx_service.py
```

返回结构按 item 分组：

```python
{
    "idx": 1,
    "item": item,
    "candidates": [],
    "provider_error": ""
}
```

候选必须写入：

- `source`
- `matched_source`
- `matched_sources`

## 3. 字段合并原则

- 采购历史字段只写采购历史数据。
- 聚水潭供应商写入 `jushuitan_supplier_name`，不要覆盖 `supplier_name`。
- 供应商报价字段使用 `supplier_quote_*` 前缀。
- 空字段不要覆盖已有有效字段。

## 4. 排序原则

优先考虑：

- 多源命中
- 有采购价
- 有供应商报价
- 有库存
- 匹配分高
- 日期新

具体逻辑以 `app/workflow/graph.py` 为准。

## 5. 风险点

- `config.py` 仍有历史默认敏感配置，生产必须用 `.env` 覆盖。
- 聚水潭接口权限变化会影响库存和动态成本。
- SQLite 报价表文件更新不是自动同步，需要替换文件并重启后端。
- 多实例部署时进程内任务和上下文不可共享。
